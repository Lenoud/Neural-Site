# 系统架构设计

## 整体架构

```
客户端 → Nginx → 前端(React) / 后端(Go)
                        ↓
                   API Gateway
                   /    |    \
              用户服务  订单服务  支付服务
                  \     |     /
                   PostgreSQL + Redis
```

## 技术选型

- 前端：React + TypeScript，参见 [[前端架构]]
- 后端：Go + gRPC，参见 [[后端规范]]
- 数据库：PostgreSQL 15
- 缓存：Redis 7
- 消息队列：Kafka
- 监控：Prometheus + Grafana

## 服务间通信

同步调用使用 gRPC，异步使用 Kafka 消息队列。
