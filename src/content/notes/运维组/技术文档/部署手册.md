# 部署手册

## 环境

| 环境 | 域名 | 用途 |
|------|------|------|
| dev | dev.example.com | 开发联调 |
| staging | staging.example.com | 预发布 |
| prod | prod.example.com | 生产 |

## 部署流程

1. 合并到 main 分支
2. CI 自动构建 Docker 镜像
3. 推送到 Harbor 仓库
4. kubectl 滚动更新
5. 健康检查通过后完成

## 回滚

```bash
kubectl rollout undo deployment/xxx
```

参见 [[发布流程]] 了解完整流程，[[2024-03-数据库连接池事故]] 了解回滚案例。
